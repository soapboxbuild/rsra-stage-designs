Design Notes Animation & Interaction Spec
Step Transitions
→ Checkmark icon uses pop-in keyframe: scale 0.5→1.15→1.0 with cubic-bezier overshoot, 350ms
→ Each step's icon animates 120ms apart when all-done state loads (staggered delay)
→ Connector line fills top-to-bottom via fill-down keyframe, 500ms ease-out, triggered after icon pop
Running State
→ Amber pulse ring: pulse-ring scale 1→1.25→1, 1.4s infinite — visible "heartbeat" for stage audience
→ Spinner ring on icon border: spin 0.9s linear infinite
→ Running badge shows small inline spinner beside "Running..." text
Streaming Results (State B)
→ Sub-panel fades up 400ms after Step 4 enters running state
→ Each metric row reveals with metric-reveal: opacity + blur(4px→0) + 4px x-shift, staggered 150ms apart
→ Skeleton bars combine shimmer + progress-bar width animation — simulate real loading
→ Dotted "Streaming results" indicator uses triple dot-pulse, offset 200ms each
State C — Completion
→ Success banner slides down with banner-in: translateY(-8px)→0 + scale(0.97)→1, 450ms cubic-bezier
→ Streaming panel transitions: skeleton bars replaced with solid green fills and real values
→ Asset header badge transitions from amber "Processing" to green "Complete"
→ In live demo: state transitions driven by polling API; use CSS class swap to trigger animations
